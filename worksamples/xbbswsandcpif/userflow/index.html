Add service

Legacy flow

The problem of creating service legacy flow
  • Once the service is created, the user is not able to change from one category to another. the buttons on the select category screen are confusing
  • In order to see the next section, the user has to input some data first
  • Not able to switch between pet types, for example, from dog to cat
  • Users are not realized they need to click the textbox to create a service
  • Creating service and add-on together is confused
  • Price and duration are separated from services. it is redundant
  • There is no way to calculate team members' capacity at this level

Wireframe

Wireframe solution
  • Add a dropdown menu with each category. Users will select a category on the first service page, afterwards, they will see a service page without category buttons on top
  • Everything shows up on one page. Some data will auto select by default with radio buttons and dropdown menu
  • Users add one service at a time, they can easily change pet type
  • Moved location to the global level, the user will not set up location here
  • Clarify the concept between add-on and service. a service must have a date, time, duration, and price. an add-on is not required. the add-on can attach to services, and also can be booked independently
  • Will add a new page for team members' availability
Next step
  • Separate input type: mandatory/additional

Wireframe

Wireframe solution

Promotion

Legacy flow

The problem of creating promotion legacy flow
  • Several screens have too much white space, squeezing them into one page is possible
  • The most important data for promotions are discount prices and the required quantity for the deal. they should be next to each other in order to adjust or compare

Wireframe

Wireframe solution
  • add quantity to service level, show the price before after discount
  • Adding expiration date feature
  • Adding service under discount setting, they can set buy x service get x service for free

Book appointment

Legacy flow

The problem of book appointment legacy flow
  • Based on the users' feedback and product managers' requirements, it needs to allow users to add more than one pet to their profile
  • Based on the interview with grooming business owners, we learned there are many add-ons will book with grooming service. We need to allow users to select add-ons with services while booking
  • Based on our competitors' research and interview with the groomer and photographers, the booking flow should allow pet parents to select a team member

Wireframe

Wireframe solution
  • Allow users to add multiple pets while adding a new client to our app
  • Able to book service with add-ons
  • Allow users to select a team member if they want to

Appointment status

Legacy flow

The problem of appointment status legacy flow
  • We need to confirm the appointment feature for business owners, here should add a confirmed status
  • While pet parents check-in, normally they fill up a form before dropping off their pet. Need a check-in status link to print form
  • Pet parents should be able to receive a notification while the appointment status update, such as appointment confirmed, appointment completed, appointment canceled

Wireframe

Wireframe solution
  • Change status "select, incomplete, pending, completed, canceled" to "pending, confirm, checked-in, in progress, completed, canceled"
  • When the client check-in, the business owner will be able to print the form for the client
  • When the appointment completed, notification will send right way
